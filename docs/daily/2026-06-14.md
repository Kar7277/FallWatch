## 今天踩过的坑 & 修复记录

### ❌ 错误 1：shm_events 和 shm_output_t 混淆
- **问题**：main.c 第 31 行写 `shm_events shm = {0}`
- **根因**：shm_events 只是事件结构体（4个int），不是带 fd/指针的管理类型
- **修复**：改为 `shm_output_t shm = {0}`
- **教训**：C 里面 struct typedef 的命名容易混淆，命名要区分"数据体"和"管理器"

### ❌ 错误 2：rgb_row 变量名不一致
- **问题**：malloc 声明 `rgb`，yuyv_to_rgb_row 写成 `rgb_row`
- **根因**：编辑时多次改名没统一
- **待修复**

### ❌ 错误 3：yuyv_to_rgb_row 只转换一行
- **问题**：函数名叫 `_row`，参数叫 `rgb_row`，每次只处理 640 像素一行
- **根因**：函数设计就是逐行的，需用 for 循环调 height 次
- **待修复**

### ❌ 错误 4：SUCCESS 日志串位
- **问题**：JPEG_SAVE_INIT SUCCESS 贴在 shm_output_init 后面
- **根因**：在 jpeg_init 和其 SUCCESS 之间插入了 shm_init
- **待修复**

### ❌ 错误 5：read_cancel 逻辑错误
- **问题**：用户点取消报警 → 打印 FAILED + 设 ret=-1 + goto cleanup
- **根因**：把正常用户操作当错误处理
- **待修复**：应该把 warning_result 复位、关 LED，不退出

## 今天学到的新知识

### ⭐ 1. POSIX 共享内存的"够用"并发
单写单读场景用 `volatile int ready` 做自旋锁就够了，不需要 pthread mutex。原因是只有一个人在写、一个人在读，不会出现写-写竞争。

### ⭐ 2. 共享内存的 deep copy 必要性
QT 拿到 mmap 指针后必须 `.copy()` 深拷贝。因为共享内存下一秒就可能被 C 端覆盖，浅拷贝会导致 QPixmap 显示时读到半帧花屏。

### ⭐ 3. 纯虚接口 vs QObject
IDataProvider 不需要继承 QObject——轮询模式不需要信号槽。让子类（LocalDataProvider）自己决定是否继承 QObject，接口保持纯 C++ 更灵活。

### ⭐ 4. Qt Designer 布局 + 代码替换的混合模式
UI 文件里用 QLabel "●" 占位，构造函数里 `removeWidget` + `insertWidget(StatusDot)` 运行时替换。好处是 Designer 里继续可视化编辑布局，代码只负责控件替换。
