# 2026-06-16 EdgeWatcher 开发日记

## NEON SIMD 加速 + SMS 调通 + 项目杀青

### NEON YUYV→RGB SIMD 加速（今日核心）

**背景**：YUYV→RGB 逐像素转换是帧率瓶颈（~14ms/帧），用 ARM NEON 128-bit SIMD 指令集加速。

**方案**：系数缩放法——将 BT.601 系数除以 8（298→37, 409→51, 100→13, 208→26, 516→65），右移 8 改为右移 5，中间值保持在 int16 范围内，避免拆半升 32 位。

**技术要点**：
- `vld4_u8` 一次加载 32 字节 YUYV 并自动解交织为 Y_even/U/Y_odd/V 四组 uint8x8_t
- `vmovl_u8` 扩展 u8→u16，`vsubq_u16` + `vdupq_n_u16` 减偏移量（Y-16, UV-128）
- `vreinterpretq_s16_u16` 转有符号，`vmulq_n_s16` + `vmlaq_n_s16` 16 位整数乘加
- `vqrshrun_n_s16` 有符号→无符号窄化 + 饱和（自带 CLAMP）
- `vzip_u8` 奇偶像素交织，`vst3_u8` 交织存出 RGB
- 每次循环处理 16 像素（8 个宏像素），640 宽正好整除

**坑**：`vqrshrn_n_s16` 返回 `int8x8_t`（-128~127），装不下 RGB 0~255，编译报 incompatible types。改为 `vqrshrun_n_s16` 修复。

### GUI 日志刷屏修复

**问题**：运动检测触发后，GUI 日志每轮询一次就追加一条"检测到运动"。

**根因**：`LocalDataProvider::getMotionStatus()` 只要 `motion==1` 就追加事件，不区分是否是新的运动事件。

**修复**：加 `m_lastMotion` / `m_lastWarning` 成员变量做上升沿/下降沿检测：
- motion 0→1：追加一次"检测到运动"
- warning 0→1：追加一次"警报已触发"
- warning 1→0：追加一次"警报已取消"

| 文件 | 变更 |
|------|------|
| `gui/edgewatcher_gui/localdataprovider.h` | 新增 `m_lastMotion`、`m_lastWarning` |
| `gui/edgewatcher_gui/localdataprovider.cpp` | `getMotionStatus()` + `getWarningStatus()` 边缘检测 |

### SMS 短信调通

**诊断过程**：
1. 确认 Air780E 串口是 `/dev/ttymxc5`（非默认的 ttymxc2）
2. `echo AT > /dev/ttymxc5` → `OK`（线没插稳，重插后正常）
3. 创建 `config.ini` 覆盖默认串口路径
4. 触发运动检测 → **真实收到短信**

| 文件 | 变更 |
|------|------|
| `src/config.c` | `uart_dev` 默认值 `ttymxc2` → `ttymxc5` |
