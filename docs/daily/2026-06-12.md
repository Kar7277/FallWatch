## 今天踩过的坑 & 修复记录

### ❌ 错误 1：大量类型错误（栈变量 vs 指针）
- **问题**：初始代码把所有模块变量声明为指针（`v4l2_capture_t *cap;`），然后到处 `*cap` 解引用野指针
- **根因**：不理解 C 语言栈分配 vs 指针的区别
- **修复**：全部改为栈变量 `v4l2_capture_t cap;`，传参用 `&cap`
- **教训**：从 ring_buffer_init/sms_alert_init 返回指针的经验错误推广到了所有模块

### ❌ 错误 2：config_load key 解析失败（前导空格）
- **问题**：config.ini 里 `  led_pin = 131`（有前导空格），key 解析为 `"  led_pin"`，strcmp 匹配不上
- **根因**：`config_load` 只用 `trim_tail_spaces` 没去掉前导空格
- **修复**：加 `trim_head_spaces` 函数，用 memmove 平移字符串
- **教训**：手写 parser 要考虑到用户编辑时的空格习惯

### ❌ 错误 3：libjpeg 版本不匹配 segfault
- **问题**：编译用 jpeglib.h v8 (API=80)，运行时链接板子系统 libjpeg.so v6b (API=62)，struct 布局不同导致段错误
- **根因**：Makefile 用 `-I` 优先找到了 buildroot sysroot 的 jpeglib.h，但 `-ljpeg` 运行时链接了板子的动态库
- **修复**：CFLAGS 加 `-I<libjpeg-turbo-3.1.4.1/build/install/include>` 放在最前面
- **教训**：静态/动态库混用时要确保头文件版本和库版本一致

### ❌ 错误 4：V4L2 未调 v4l2_start 导致 POLLERR
- **问题**：capture_thread_start 后 poll 报 POLLERR/POLLHUP
- **根因**：main.c 里只调了 `v4l2_init`，漏掉了 `v4l2_start`（VIDIOC_STREAMON）
- **修复**：在 `capture_thread_start` 前加 `v4l2_start(&cap)`
- **教训**：V4L2 五步流程（init→start→loop→stop→destroy）缺一不可

### ❌ 错误 5：GPIO led_pin 写成了按钮引脚
- **问题**：led_pin=129 是 User1 Button（gpio-keys 占用），无法 export
- **根因**：不清楚 GPIO 编号和物理引脚的对应关系
- **修复**：GPIO5_IO03 = 128+3 = 131，设备树里 status="disabled" 空闲可用
- **教训**：先查 `cat /sys/kernel/debug/gpio` 确认引脚状态再配置

## 今天学到的新知识

### ⭐ 1. INI 文件解析
- sscanf 的 `%[^=]` 和 `%[^\n]` 扫描集用法
- 键值对解析 + 段名切换状态机模式
- 前导/尾随空格处理（`trim_head_spaces` + `trim_tail_spaces`）

### ⭐ 2. Makefile 基础
- `$(wildcard *.c)` 自动收集源文件
- `$(SRCS:.c=.o)` 模式替换
- `$<` `$@` `$^` 自动变量
- 静态库直接写路径 vs `-L` + `-l`

### ⭐ 3. libjpeg 版本号机制
- `JPEG_LIB_VERSION` 宏：v6b=62, v8=80, v9=90
- libjpeg-turbo 3.1.x 兼容 v6b API (62)
- `strings libjpeg.a | grep version` 查版本

### ⭐ 4. 设备树 GPIO 调试
- `cat /sys/kernel/debug/gpio` 查看所有 GPIO 状态
- gpiochip 基地址换算：gpio5 = gpiochip128
