## 今天踩过的坑 & 修复记录

### ❌ 错误 1：make_yuyv_frame NULL 检查在解引用之后

- **问题**：先 `frame->id=0`，再 `if(frame==NULL)`
- **根因**：疏忽，malloc 后的检查必须紧跟在 malloc 之后
- **修复**：NULL 检查移到所有解引用之前
- **教训**：malloc 返回值检查是第一优先级，不能有任何代码插在中间

### ❌ 错误 2：循环只填充了一半缓冲区

- **问题**：`for(i=0; i<total_pixels/2; i+=2)` 只填了 `w*h` 字节，缓冲区是 `w*h*2`
- **根因**：变量名 `total_pixels` 存的是 `data_size`（字节数），名字误导，上界写了 `/2`
- **修复**：删掉 `/2`，变量改名为 `data_size`
- **教训**：变量命名就是文档——叫 `total_pixels` 就只存像素数，别存字节数

### ❌ 错误 3：set_y_region 循环条件错误

- **问题**：`for(i=start; i<count; i++)` → `set_y_region(f, 10, 5, 100)` 时 `10<5` 直接退出
- **根因**：把"数量"当成"结束位置"
- **修复**：`i < start + count`
- **教训**：循环边界 = 起始 + 数量，不是直接用数量做上界

### ❌ 错误 4：变量名重复定义

- **问题**：Group 1 和 Group 2 都声明了 `motion_detect_t md`
- **修复**：Group 2 改名为 `md2`
- **教训**：多个测试组之间要么用大括号隔离作用域，要么用不同变量名

### ❌ 错误 5：4b CHECK 期望值/描述写错

- **问题**：AND=1 全像素运动场景，CHECK 期望 `== 0` 且描述写 "no motion"
- **修复**：改成 `== 1`，描述改为 "AND=1: all pixels -> 1"
- **教训**：测试用例的期望值要和设计文档一一对照

### ❌ 错误 6：4d 标签/注释错误

- **问题**：标签写"阈值过滤"（应该是"比例过滤"），注释说"前半区域无运动"（实际前半区域 AND=1）
- **修复**：标签改正确，注释反转
- **教训**：复制粘贴后必须逐个字段改干净

### ❌ 错误 7：拼写错误

- **问题**：`"no crush"`（crush=压碎）
- **修复**：`"no crash"`（crash=崩溃）
- **教训**：英文拼写检查不能省

### ❌ 错误 8：缺少 main() 闭合大括号

- **问题**：Group 5 的 `}` 闭合了块，但 main() 函数体没有闭合
- **修复**：文件末尾加 `}`

## 今天学到的新知识

### ⭐ 1. V4L2 bytesused vs buf.length

- `buf.length`：缓冲区总容量（mmap/munmap 时用）
- `buf.bytesused`：这一帧实际有效字节数（DQBUF 后读取和构造 frame_t 时用）
- 对于 YUYV 裸流两者通常相等，但 MJPEG 等压缩格式差异很大
- 语义上 `bytesused` 才是正确选择

### ⭐ 2. 三帧差法 AND 逻辑细节

- diff1 = |prev2 - prev1|，diff2 = |prev1 - curr|
- 逐像素 AND：两个 diff 都 > threshold 才计入 motion_pixels
- 差分结果不缓存，相邻两次 process 会重复计算，但额外开销仅 0.3%

### ⭐ 3. CHECK 宏 do-while(0) 模式

- 包在 `do { ... } while(0)` 里让多语句宏像一个普通语句
- 如果不用 do-while 包起来，`if-else` 配对会出语法问题
- 宏内部直接引用 main() 里的 `passed`/`total` 局部变量

### ⭐ 4. Linux 串口编程 termios（初步了解）

- `struct termios` 配置波特率、数据位、停止位、校验位
- `cfsetispeed`/`cfsetospeed` 设置波特率
- `tcsetattr` 应用配置

### ⭐ 5. AT 指令发短信流程

- `AT+CMGF=1` → 文本模式
- `AT+CMGS="号码"` → 设收件人
- 模块回 `>` 后发短信内容，`\x1A` 结束
