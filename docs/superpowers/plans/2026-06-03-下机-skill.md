# /下机

收到 `/下机` 指令后，严格按顺序执行以下三个任务。

## 任务 1：写开发日记

1. **回顾本次对话**，提取今天完成的开发工作：
   - 新建/修改了哪些文件
   - 通过了哪些测试（测试名 + 通过数/总数 + 关键指标如帧率）
   - 修复了哪些 bug（问题 → 根因 → 修复 → 教训）
   - 学到了哪些新知识
2. **按以下模板写入** `daily/YYYY-MM-DD.md`（日期用今天的实际日期）：

```markdown
# YYYY-MM-DD EdgeWatcher 开发日记

## 今日任务总览
（一段话概括今天做了什么）

## 完成内容
（每个模块一个 ### 三级标题 + 表格记录文件变更）

## 今天踩过的坑 & 修复记录
（每个错误：### ❌ 错误 N：标题 + 问题/根因/修复/教训 四项）

## 今天学到的新知识
（### ⭐ N. 知识点标题 + 说明）

## 当前文件结构
（do_my_test/ 目录树，用 ✅ ⬜ 🟡 标记状态）

## 下一步
（编号列表）
```

3. 日记必须包含实际内容，不允许写"TBD"或"略"。

## 任务 2：全量同步记忆

### 2a. 更新 memory 文件

## 执行规则

1. **严格顺序执行**：任务 1 → 任务 2 → 任务 3，不可并行
2. **错误不阻断**：任一任务失败，报告错误后继续执行后续任务
3. **最后汇总**：三个任务完成后，输出汇总报告：
   ```
   📋 /下机 完成：
   ✅ 开发日记 → daily/YYYY-MM-DD.md
   ✅ 同步记忆 → 更新了 X 个 memory 文件
   ✅ codegraph sync → 完成
   （如果有失败的用 ❌ 标记）
   ```
```
