# /下机 Skill 设计文档

**日期：** 2026-06-03
**类型：** Skill 开发
**状态：** 已批准

---

## 1. 目标

创建一个 `/下机` skill，用一个命令完成每次退出前的三个收尾任务：

1. **写开发日记** — 总结今日对话，写入 `daily/YYYY-MM-DD.md`
2. **全量同步记忆** — 更新 memory 文件 + 整理 CLAUDE.md + 清理过时记忆
3. **同步 codegraph** — 执行 `codegraph sync` 更新项目索引

## 2. 文件位置

```
EdgeWatcher/
└── .claude/
    └── skills/
        └── 下机.md          ← 新建
```

## 3. 任务 1：写开发日记

### 3.1 触发逻辑

回顾本次 session 的对话内容，提取开发活动信息。

### 3.2 提取内容

- 完成了什么（新文件、新功能、测试结果）
- 踩了什么坑（bug + 根因 + 修复方法 + 教训）
- 学到了什么（新概念、新认知、新技巧）
- 文件结构变化（新增/修改的文件）

### 3.3 输出格式

写入 `daily/YYYY-MM-DD.md`，沿用现有模板：

```markdown
# YYYY-MM-DD EdgeWatcher 开发日记

## 今日任务总览
（一段话概括）

## 完成内容
（每个模块一个 ### 三级标题 + 表格记录文件变更）

## 今天踩过的坑 & 修复记录
（每个错误：### ❌ 错误 N + 问题/根因/修复/教训）

## 今天学到的新知识
（### ⭐ N. 知识点 + 是什么/为什么重要/怎么用）

## 当前文件结构
（do_my_test/ 目录树）

## 下一步
（编号列表）
```

## 4. 任务 2：全量同步记忆

### 4a. 更新 memory 文件

扫描本次对话，按分类写入 `memory/` 目录：

| 类型 | 触发条件 | 操作 |
|------|----------|------|
| 新概念 | 今天学到的新知识 | 新建 memory 文件 |
| 踩坑记录 | 今天遇到的 bug | 追加到已有文件或新建 |
| 项目进度 | 模块状态变化 | 更新 `project_edgewatcher.md` |
| 目录结构 | 文件增删 | 更新 `project_edgewatcher_dir.md` |
| 用户偏好 | 新工作习惯 | 更新 `user_communication_rules.md` |

### 4b. 整理 CLAUDE.md

- 检查项目根目录 `CLAUDE.md` 是否存在
- 把今天新增的重要约定、注意事项同步进去
- 不存在则跳过

### 4c. 清理过时记忆

- 扫描 memory 文件，找和当前事实矛盾的内容
- 修正或删除矛盾条目
- 不做大范围重构，只修正明显错误

## 5. 任务 3：同步 codegraph

- 工作目录：`C:\desk\project things\EdgeWatcher`
- 执行命令：`codegraph sync`
- 功能：增量扫描新增/修改文件，更新 `.codegraph/codegraph.db`

## 6. 执行顺序

```
/下机
 │
 ├─ ① 写开发日记（任务 1）
 │
 ├─ ② 全量同步记忆（任务 2）
 │
 └─ ③ codegraph sync（任务 3）
```

严格顺序执行，不并行（任务 1 和 2 都依赖对话内容回顾）。

## 7. 错误处理

- 任务 1 失败（如无法写入文件）：报告错误但继续执行后续任务
- 任务 2 失败（如 memory 目录不存在）：报告错误但继续执行
- 任务 3 失败（如 codegraph 未安装）：报告错误
- 每个任务独立报告结果，最后汇总

## 8. 不在范围内的内容

- 不自动 git commit（由用户自行决定）
- 不关机/不退出终端
- 不做项目外的事情
