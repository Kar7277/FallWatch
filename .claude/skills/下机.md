# /下机

收到 `/下机` 指令后，严格按顺序执行以下三个任务。

## 任务 1：写开发日记

1. **回顾本次对话**，提取今天完成的开发工作：
   - 新建/修改了哪些文件
   - 通过了哪些测试（测试名 + 通过数/总数 + 关键指标如帧率）
   - 修复了哪些 bug（问题 → 根因 → 修复 → 教训）
   - 学到了哪些新知识
2. **按以下模板写入** `daily/YYYY-MM-DD.md`（日期用今天的实际日期）：

```markdown
# YYYY-MM-DD EdgeWatcher 开发日记

## 今日任务总览
（一段话概括今天做了什么）

## 完成内容
（每个模块一个 ### 三级标题 + 表格记录文件变更）

## 今天踩过的坑 & 修复记录
（每个错误：### ❌ 错误 N：标题 + 问题/根因/修复/教训 四项）

## 今天学到的新知识
（### ⭐ N. 知识点标题 + 说明）

## 当前文件结构
（do_my_test/ 目录树，用 ✅ ⬜ 🟡 标记状态）

## 下一步
（编号列表）
```

3. 日记必须包含实际内容，不允许写"TBD"或"略"。

## 任务 2：全量同步记忆

### 2a. 更新 memory 文件

扫描本次对话，按以下规则更新 memory 目录（默认为用户级 memory 路径）：

| 情况 | 操作 |
|------|------|
| 今天学到全新概念/技术 | 新建 `concept_xxx.md` |
| 今天踩了新的坑 | 追加到对应模块的 memory 或新建 |
| 模块开发状态变化（⬜→✅） | 更新项目进度 memory 的"当前开发进度"表格 |
| 文件有增删 | 更新项目目录结构 memory 的文件树 |
| 新的工作习惯/约定 | 更新用户沟通规则 memory |

### 2b. 整理 CLAUDE.md

- 检查项目根目录 `CLAUDE.md` 是否存在
- 如果存在且今天有新的约定/注意事项，同步进去
- 不存在则跳过

### 2c. 清理过时记忆

- 快速扫描 memory 文件，找和当前事实矛盾的内容
- 修正或删除明显过时的条目
- 不做大范围重构

### 2d. 更新 MEMORY.md 索引

- 如果有新增/删除 memory 文件，更新 `MEMORY.md` 的索引列表

## 任务 3：同步 codegraph

执行以下命令：

```bash
codegraph sync
```

> 需要在已初始化 codegraph 的项目根目录下执行。

## 执行规则

1. **严格顺序执行**：任务 1 → 任务 2 → 任务 3，不可并行
2. **错误不阻断**：任一任务失败，报告错误后继续执行后续任务
3. **最后汇总**：三个任务完成后，输出汇总报告：

```
📋 /下机 完成：
✅ 开发日记 → daily/YYYY-MM-DD.md
✅ 同步记忆 → 更新了 X 个 memory 文件
✅ codegraph sync → 完成
```

（如果有失败的步骤用 ❌ 标记）
